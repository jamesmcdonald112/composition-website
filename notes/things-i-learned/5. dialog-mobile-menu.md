# Mobile menu with the native <dialog> element

The browser's native `<dialog>` element gives you a focus trap, Escape key handling, and scroll lock for free — no JS needed for those.

```html
<dialog data-nav-mobile aria-label="Navigation menu">
  <!-- menu content -->
</dialog>
```

```ts
const toggle = document.querySelector<HTMLButtonElement>("[data-navbar-toggle]");
const dialog = document.querySelector<HTMLDialogElement>("[data-nav-mobile]");
const closeBtn = document.querySelector<HTMLButtonElement>("[data-nav-mobile-close]");

const openMenu = () => {
  toggle.setAttribute("aria-expanded", "true");
  dialog.showModal(); // opens as modal — browser handles focus trap + Escape
};

const closeMenu = () => {
  toggle.setAttribute("aria-expanded", "false");
  dialog.close(); // browser restores focus to the toggle automatically
};

toggle.addEventListener("click", openMenu);
closeBtn.addEventListener("click", closeMenu);

// Sync aria-expanded when browser closes via Escape key
dialog.addEventListener("close", () => {
  toggle.setAttribute("aria-expanded", "false");
});
```

**What the browser gives you for free with `showModal()`:**
- Focus is trapped inside the dialog — tab key can't escape
- Pressing Escape closes it automatically
- Background scroll is locked
- Focus returns to the trigger button when closed

**Why `aria-expanded`:** Screen readers use this to announce whether the menu is open or closed. Always keep it in sync with the actual state.
