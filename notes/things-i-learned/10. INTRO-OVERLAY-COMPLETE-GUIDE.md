# Intro Overlay ŌĆö Complete Guide

How to build a full-screen intro/preloader animation in Astro with GSAP,
ensure it only plays once per session, and avoid every pitfall we hit.

---

## What It Does

A full-screen dark navy overlay appears on first visit. The firm logo,
a gold rule, and the firm name animate in, hold briefly, then the overlay
wipes upward off the screen revealing the page beneath. On every subsequent
page visit within the same browser session, the overlay is skipped entirely.

## How It Works ŌĆö The Three Layers

### Layer 1 ŌĆö Prevent the navy flash before JS runs

The overlay CSS paints the dark navy background immediately when the browser
parses the HTML. JS runs after. If the intro has already been seen, you get
a brief navy flash before JS removes the element.

Fix: add an `is:inline` script as the **very first element in `<head>`**,
before any stylesheets. It runs synchronously before paint.

```html
<!-- BaseLayout.astro ŌĆö must be FIRST in <head> -->
<script is:inline>
  if (sessionStorage.getItem('intro-seen')) {
    document.documentElement.classList.add('intro-done');
  }
  document.addEventListener('astro:after-swap', function() {
    if (sessionStorage.getItem('intro-seen')) {
      document.documentElement.classList.add('intro-done');
    }
  });
</script>
```

The `astro:after-swap` listener handles ClientRouter soft navigations ŌĆö
it fires before paint on each navigation so the class is always applied
before anything renders.

### Layer 2 ŌĆö Hide via CSS before paint

In `Intro.astro`, add this CSS so that when `intro-done` is on `<html>`,
the overlay never renders at all:

```css
:global(.intro-done) [data-intro] {
  display: none !important;
}
```

`display: none` is instant ŌĆö the browser never paints the element.

Also add `visibility: hidden` to the animated elements so they don't
flash at their final state before GSAP applies initial values:

```css
.intro__mark,
.intro__name,
.intro__sub {
  opacity: 0;
  visibility: hidden;
}
```

GSAP then sets `visibility: visible` as its first action before animating.

### Layer 3 ŌĆö The animation script

`intro.anim.ts` runs immediately on script execution ŌĆö no event listener.
Astro module scripts run once on every hard page load automatically.

```ts
import { gsap } from "../../scripts/gsap.ts";

const intro = document.querySelector<HTMLElement>("[data-intro]");
if (!intro) throw new Error("Intro: [data-intro] not found");

if (sessionStorage.getItem("intro-seen")) {
  intro.remove();
} else {
  sessionStorage.setItem("intro-seen", "1");

  if (window.matchMedia("(prefers-reduced-motion: reduce)").matches) {
    intro.remove();
  } else {
    // GSAP animation runs here
  }
}
```

## Why `sessionStorage` and Not `localStorage`

`sessionStorage` persists for the duration of a browser tab session.
Closing the tab or browser clears it, so the intro plays again on the
next visit. This is intentional ŌĆö fresh each visit, not intrusive within
a session.

`localStorage` would mean the intro never plays again even after closing
the browser, which is too aggressive.

## Every Problem We Hit and How We Fixed It

### Problem 1 ŌĆö Navy flash on pages after the first visit

**What happened:** The intro CSS painted the dark navy background before
JS ran. Even when the session flag was set, there was a visible flash.

**Fix:** The `is:inline` script in `<head>` (Layer 1) runs before CSS
and adds `intro-done` to `<html>`. The CSS rule `display: none` on
`[data-intro]` then hides the element before it ever paints.

---

### Problem 2 ŌĆö Intro replayed after form submission redirect

**What happened:** After a successful contact form submission, the browser
does a POST ŌåÆ 302 redirect ŌåÆ GET `/thank-you`. This is a full page reload.
The intro script ran fresh on the thank-you page and played again.

**Fix:** The `is:inline` script in `<head>` catches this. On the thank-you
page hard load, `sessionStorage` already has `intro-seen`, so `intro-done`
is added to `<html>` before paint and the overlay never appears.

---

### Problem 3 ŌĆö `astro:page-load` does not fire on hard loads reliably

**What happened:** We tried wrapping the animation in `astro:page-load`
thinking it fires on both hard loads and soft navigations. On some pages
(specifically the home page loaded directly) it did not fire, leaving the
navy overlay permanently on screen.

**Fix:** Don't use `astro:page-load` for the animation at all. Astro module
scripts execute immediately on hard page loads ŌĆö no listener needed. The
script just runs at the top level and the animation fires instantly.

---

### Problem 4 ŌĆö `astro:page-load` fired twice on `/contact`

**What happened:** The contact page has `<ClientRouter />` from the form
component. This caused `astro:page-load` to fire twice on that page's hard
load. The second firing found `intro-seen` already set and removed the intro
element mid-animation.

**Fix:** Same as Problem 3 ŌĆö remove `astro:page-load` entirely from the
animation script. Immediate execution at the top level only fires once.

---

### Problem 5 ŌĆö `astro:after-swap` in the inline script caused the animation to never play

**What happened:** We added `astro:after-swap` to the inline script thinking
it would help with ClientRouter navigations. But this caused `intro-done` to
be applied on the very first load in some cases, suppressing the animation
before GSAP could run.

**Fix:** `astro:after-swap` only fires on ClientRouter soft navigations, not
on the initial hard load. So it is safe in the inline script ŌĆö it will never
run on first load, only on subsequent navigations. The bug was caused by
something else at the time. With the correct final setup it works fine.

## The Final Working Setup Summary

| Layer | File | What it does |
|---|---|---|
| Inline `<head>` script | `BaseLayout.astro` | Adds `intro-done` to `<html>` before paint if session flag is set. Also listens for `astro:after-swap` for soft navigations. |
| CSS hide rule | `Intro.astro` | `display: none` on `[data-intro]` when `intro-done` is present |
| `visibility: hidden` on elements | `Intro.astro` | Prevents GSAP flash ŌĆö elements hidden until GSAP sets them visible |
| Animation script | `intro.anim.ts` | Runs immediately on execution. Checks flag, removes or animates. No event listener. |
| Session flag | `sessionStorage` | `intro-seen` key. Set once. Cleared when browser/tab is closed. |

---

## What NOT to Do

- **Do not use `astro:page-load`** for the animation ŌĆö it is unreliable on
  hard loads and can fire multiple times on pages with `ClientRouter`
- **Do not use `localStorage`** ŌĆö the intro will never replay even after
  the browser is closed, which feels broken
- **Do not put the flash-prevention script anywhere other than the first
  element in `<head>`** ŌĆö it must run before stylesheets parse or it is useless
- **Do not add `astro:after-swap` to the animation script** ŌĆö only belongs
  in the inline `<head>` script for the CSS class toggle
