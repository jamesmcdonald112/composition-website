# Intro Replay Fix — POST Redirect to Thank-You Page

## The Problem

After a successful contact form submission, the browser was redirected from `/contact` to `/thank-you`. On the thank-you page, the full intro/preloader animation played again — showing a dark navy screen before the page content appeared.

## Why It Happened

The form uses a PRG (Post/Redirect/Get) pattern:

1. User submits form → `POST /contact`
2. Middleware handles the action → `302 redirect` to `/thank-you`
3. Browser follows redirect → `GET /thank-you` (full page reload)

Because step 3 is a **full page reload**, the intro script ran from scratch every time. The fix needed two things: hide the intro element before CSS paints, and ensure the script runs at the right moment in the Astro lifecycle.

## The Fix — Three Changes

### 1. Inline script at top of `<head>` in `BaseLayout.astro`

The intro's CSS (dark navy full-screen overlay) was painting before JS ran — causing a visible blue flash before the intro removed itself. The fix: add an `is:inline` script as the **first thing in `<head>`** so it runs before any CSS is parsed. It adds a class to `<html>` if the intro has already been seen.

```html
<script is:inline>
  if (sessionStorage.getItem('intro-seen')) {
    document.documentElement.classList.add('intro-done');
  }
</script>
```

### 2. CSS in `Intro.astro` to hide immediately when class is present

```css
:global(.intro-done) [data-intro] {
  display: none !important;
}
```

When `intro-done` is on `<html>`, the intro element is hidden before it ever paints. No flash.

### 3. Wrap animation logic in `astro:page-load` in `intro.anim.ts`

The original script ran immediately on script execution. Wrapping it in `astro:page-load` ensures it fires correctly for both full page loads and ClientRouter soft navigations.

Also replaced the `throw new Error` for a missing intro with an early `return` — the intro element only exists on pages that include the component, so a missing element is not an error.

```ts
document.addEventListener("astro:page-load", () => {
  const intro = document.querySelector("[data-intro]");
  if (!intro) return;

  if (sessionStorage.getItem("intro-seen")) {
    intro.remove();
    return;
  }

  sessionStorage.setItem("intro-seen", "1");
  // ... rest of animation unchanged
});
```

## Storage choice — `sessionStorage`

The intro uses `sessionStorage` so it plays once per browser session. Closing and reopening the browser or tab will show it again. This is intentional — it feels fresh on each visit without being intrusive within a single session.

## Files Changed

| File | Change |
|---|---|
| `src/components/Intro/intro.anim.ts` | Wrapped in `astro:page-load`, early return instead of throw |
| `src/layouts/BaseLayout.astro` | Added `is:inline` script as first `<head>` element |
| `src/components/Intro/Intro.astro` | Added `:global(.intro-done) [data-intro] { display: none }` CSS |
