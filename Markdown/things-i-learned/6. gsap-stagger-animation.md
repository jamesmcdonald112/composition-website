# GSAP stagger animation for list items

Animating a list of elements so they cascade in one by one rather than all at once.

```ts
import { gsap } from "../../scripts/gsap";

// Always guard for reduced motion first
const reducedMotion = window.matchMedia("(prefers-reduced-motion: reduce)").matches;

if (!reducedMotion) {
  gsap.fromTo(
    Array.from(navLinks),   // convert NodeList to array so GSAP can iterate it
    { y: 40, opacity: 0 }, // start state — below and invisible
    {
      y: 0,                 // end state — normal position
      opacity: 1,           // end state — fully visible
      stagger: 0.08,        // each item starts 0.08s after the previous one
      duration: 0.7,
      ease: "power3.out",
      delay: 0.1,           // wait 0.1s before starting (lets the panel open first)
    },
  );
}
```

**Key points:**
- `fromTo` is explicit — you define both start and end states. Use this when you need predictable results (e.g. re-running the animation on menu open)
- `stagger` is what creates the cascade effect — without it all items animate simultaneously
- `Array.from()` is needed because `querySelectorAll` returns a NodeList, not an array
- Always check `prefers-reduced-motion` before running any animation — users who have this set find motion uncomfortable or disorienting
