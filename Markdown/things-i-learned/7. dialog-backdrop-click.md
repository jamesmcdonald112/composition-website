# Detecting a click on the dialog backdrop

When using `showModal()`, clicking outside the dialog (on the `::backdrop`) doesn't fire a close event by default. You have to detect it manually.

```ts
dialog.addEventListener("click", (e) => {
  const rect = dialog.getBoundingClientRect();
  const clickedOutside =
    e.clientX < rect.left ||
    e.clientX > rect.right ||
    e.clientY < rect.top ||
    e.clientY > rect.bottom;

  if (clickedOutside) closeMenu();
});
```

**Why this works:**
- Clicks on the `::backdrop` still register on the `<dialog>` element itself
- `getBoundingClientRect()` gives you the position and size of the dialog content area
- If the click coordinates fall outside that rectangle, it must have been on the backdrop

**Why not just listen on `::backdrop` directly:** You can't attach event listeners to pseudo-elements in JS — they don't exist as real DOM nodes.
