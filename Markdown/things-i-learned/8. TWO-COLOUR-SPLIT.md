# Two-Colour Full-Height Page Split

The technique used on the contact page. A two-panel layout where each
background colour bleeds to the viewport edge, while the content stays
inside the site wrapper — aligned with every other section on the site.

---

## The problem it solves

If you put two coloured divs side by side inside a max-width wrapper,
the backgrounds stop at the wrapper edge. You get a floating two-tone
box in the middle of the page — not a full-bleed split.

The fix: use `::before` pseudo-elements to extend each background
outward beyond the wrapper, while the content stays put.

---

## Page file structure

```astro
<BaseLayout>
  <div class="contact-page wrapper wrapper--wide">
    <LeftPanel />
    <RightPanel />
  </div>
</BaseLayout>
```

`.wrapper.wrapper--wide` handles max-width (90rem), centring, and
responsive horizontal padding — no extra container needed. Content
aligns with the navbar and hero automatically.

```css
.contact-page {
  display: grid;
  grid-template-columns: 1fr;
  min-height: 100vh;
  position: relative;          /* required — panels use position: absolute on ::before */

  @media (min-width: 768px) {
    grid-template-columns: 42fr 58fr;   /* adjust ratio to taste */
  }
}
```

---

## Left panel — extend background leftward

```css
.left-panel {
  position: relative;
  background: var(--color-bg-brand);
  padding-block: var(--space-5xl);
  padding-inline-end: var(--space-3xl);
  /* no padding-inline-start — the wrapper's own padding provides it */
}

.left-panel::before {
  content: '';
  position: absolute;
  inset-block: 0;
  inset-inline-end: 100%;   /* starts at the left edge of the panel */
  width: 50vw;              /* reaches the viewport edge */
  background: var(--color-bg-brand);
  pointer-events: none;
}
```

## Right panel — extend background rightward

```css
.right-panel {
  position: relative;
  background: var(--color-bg-surface);
  padding-block: var(--space-5xl);
  padding-inline-start: var(--space-3xl);
  /* no padding-inline-end — the wrapper's own padding provides it */
}

.right-panel::before {
  content: '';
  position: absolute;
  inset-block: 0;
  inset-inline-start: 100%;  /* starts at the right edge of the panel */
  width: 50vw;               /* reaches the viewport edge */
  background: var(--color-bg-surface);
  pointer-events: none;
}
```

---

## Why this works

- The wrapper clips the content, not the backgrounds.
- `::before` sits outside the wrapper in absolute space, so it
  reaches the viewport edge regardless of the wrapper's max-width.
- Each panel only needs to extend in one direction — left panel goes
  left, right panel goes right. Together they cover the full viewport.
- On mobile (single column), the `::before` pseudo-elements are
  harmless — they extend sideways but the panel fills full width
  anyway so there's nothing to see beyond the edges.

---

## Which wrapper to use

Use `wrapper wrapper--wide` (90rem / 1440px) if the page has a
full-height or hero-scale feel — matches the navbar and hero width.

Use `wrapper` alone (72rem / 1152px) if the split panel is more
like a content section alongside other sections on the page.

---

## The rule about padding

Each panel takes padding from two sources:

| Side | Source |
|---|---|
| Outer edge (left of left panel, right of right panel) | `.wrapper` padding-inline |
| Inner edge (where panels meet) | Panel's own `padding-inline-start` or `padding-inline-end` |

Never add padding on the outer edge of either panel — the wrapper
already provides it, and doubling it breaks alignment with the navbar.
