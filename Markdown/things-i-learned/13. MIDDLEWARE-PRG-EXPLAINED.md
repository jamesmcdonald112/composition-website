# How the Middleware Works — PRG Pattern Explained

This is a line-by-line breakdown of `src/middleware/index.ts`.
Middleware runs on every request before Astro renders the page.

---

## Imports

```ts
import { getActionContext, type SerializedActionResult } from "astro:actions";
import { defineMiddleware } from "astro:middleware";
```

Pull in Astro's built-in tools. `getActionContext` gives access to whatever form action was submitted. `defineMiddleware` is the wrapper that makes this file recognised as middleware by Astro.

---

## Constants

```ts
const ACTION_RESULT_KEY = "action-result";
const FORM_DATA_KEY = "form-data";
const ACTION_RESULT_TTL_SECONDS = 60;
```

Named keys for what gets stored in the session, and how long it lives. 60 seconds is enough for the redirect round-trip — nobody needs this data after that.

---

## Type

```ts
type StoredActionResult = {
  actionName: string;
  actionResult: SerializedActionResult;
};
```

A TypeScript shape describing what gets saved to the session. Keeps the code type-safe when reading back from the session.

---

## Middleware definition

```ts
export const onRequest = defineMiddleware(async (context, next) => {
```

This function runs on every incoming request before Astro renders the page. `context` is the request. `next` moves on to normal page rendering.

---

## Skip prerendered routes

```ts
if (context.isPrerendered) {
  return next();
}
```

Static pages are built at compile time — sessions and actions do not apply to them. Skip immediately.

---

## Get action context

```ts
const { action, setActionResult, serializeActionResult } = getActionContext(context);
```

Checks if this request is a form submission to an Astro action. `action` contains what was submitted. `setActionResult` lets you inject a result back so the page can read it via `Astro.getActionResult()`. `serializeActionResult` converts the result to a format safe for session storage.

---

## Get session

```ts
const session = context.session;
if (!session) {
  return next();
}
```

Grab the session object. If sessions are unavailable (some edge runtimes do not support them), bail out safely.

---

## Read flash data from previous request

```ts
const stored = await session.get<StoredActionResult>(ACTION_RESULT_KEY);
if (stored) {
  setActionResult(stored.actionName, stored.actionResult);
  session.delete(ACTION_RESULT_KEY);
}
```

This runs on the GET after a redirect. If the previous POST saved an action result, restore it so the page can display error messages via `Astro.getActionResult()`. Delete it immediately — flash data is use-once. This is the core of the PRG pattern.

---

## Intercept form POST

```ts
if (action?.calledFrom === "form") {
```

Only proceed if this is an HTML form submission to an Astro action, not a regular page load.

---

## Capture form values before running the action

```ts
const rawFormData = await context.request.clone().formData();
const formValues: Record<string, string> = {};
for (const [key, value] of rawFormData.entries()) {
  if (typeof value === "string") formValues[key] = value;
}
```

Clone the request before reading it — request bodies can only be read once. Extract all the text fields into a plain object. File uploads are skipped (`typeof value === "string"` filters them out). These values will be used to repopulate form fields if validation fails.

---

## Run the action

```ts
const actionResult = await action.handler();
```

Actually execute the form action — this is where Zod validation and your business logic run.

---

## Save the result to session

```ts
session.set(ACTION_RESULT_KEY, {
  actionName: action.name,
  actionResult: serializeActionResult(actionResult),
}, { ttl: ACTION_RESULT_TTL_SECONDS });
```

Save the result (success or error) to the session so the next GET request can read it. TTL of 60 seconds means it auto-expires if the redirect never happens.

---

## Handle validation errors

```ts
if (actionResult.error) {
  session.set(FORM_DATA_KEY, formValues, { ttl: ACTION_RESULT_TTL_SECONDS });
  const referer = context.request.headers.get("referer");
  if (referer) {
    return context.redirect(referer);
  }
}
```

If the action failed, also save the raw form values so fields can be repopulated. Then redirect back to the form page. `referer` is the URL the user came from — almost always `/contact`. This is the POST → Redirect part of PRG.

---

## Handle success

```ts
return context.redirect("/thank-you");
```

If no error, redirect to the thank-you page. This is the GET of PRG on the happy path.

---

## Non-action requests

```ts
return next();
```

Anything that is not a form action POST — regular page loads, API calls — just passes through normally.

---

```
   ↙      ↘

↓

```
