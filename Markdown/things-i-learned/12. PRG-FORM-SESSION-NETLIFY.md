# PRG Form Pattern — Session Persistence on Netlify

How to correctly carry form field values across a POST → redirect → GET cycle in Astro 6 SSR on Netlify, without ClientRouter.

---

## The Problem

Without ClientRouter, every redirect is a full page reload. When a user submits the form with invalid data:

1. Browser POSTs to `/contact`
2. Middleware catches it, runs the action, gets a validation error
3. Middleware redirects back to `/contact`
4. Browser GETs `/contact` — form shows errors but **fields are empty**

The user has to re-type everything.

---

## Why `sessionDrivers.memory()` Does Not Work

`sessionDrivers.memory()` stores data in process memory. On Netlify, each serverless function invocation is its own process. The POST and the subsequent GET redirect hit **different function instances** — the memory from the POST is gone by the time the GET arrives.

This is why session data set during the POST never appeared on the GET.

---

## The Fix — Remove the Manual Session Driver

The Netlify adapter **automatically configures Netlify Blobs** as the session driver. Netlify Blobs is a persistent key-value store that survives across function invocations — exactly what PRG needs.

Manually specifying `sessionDrivers.memory()` overrides this correct default with a broken one.

**`astro.config.mjs` — before (broken):**
```js
import { defineConfig, envField, sessionDrivers } from "astro/config";

export default defineConfig({
  session: {
    driver: sessionDrivers.memory(),
  },
});
```

**`astro.config.mjs` — after (correct):**
```js
import { defineConfig, envField } from "astro/config";

export default defineConfig({
  // No session config — Netlify adapter uses Netlify Blobs automatically
});
```

## The Full Pattern

### Middleware — store form values on error

```ts
if (action?.calledFrom === "form") {
  // Capture raw form data before handler runs
  const rawFormData = await context.request.clone().formData();
  const formValues: Record<string, string> = {};
  for (const [key, value] of rawFormData.entries()) {
    if (typeof value === "string") formValues[key] = value;
  }

  const actionResult = await action.handler();

  session.set("action-result", { actionName, actionResult: serializeActionResult(actionResult) }, { ttl: 60 });

  if (actionResult.error) {
    // Store form values — survives the redirect via Netlify Blobs
    session.set("form-data", formValues, { ttl: 60 });
    return context.redirect(referer ?? "/contact");
  }

  return context.redirect("/thank-you");
}
```

### Component — read and repopulate

```ts
const result = Astro.getActionResult(actions.contact);
const error = result?.error;
const fieldErrors = isInputError(error) ? error.fields : {};

// Read stored values — only exists on error redirect
const prev = error ? await Astro.session?.get<Record<string, string>>("form-data") : null;
if (prev) await Astro.session?.delete("form-data");
```

Then in the template:
```astro
<input name="name" value={prev?.name} />
<input name="email" value={prev?.email} />
<textarea name="message">{prev?.message}</textarea>
<option value={area} selected={prev?.service === area}>{area}</option>
```

---

## Key Rules

- Never set `sessionDrivers.memory()` when deploying to Netlify — it breaks cross-request persistence
- The Netlify adapter handles session automatically — trust it
- Store form values in the session, not a cookie — the session is the right transport layer
- Delete the `form-data` key after reading it — flash semantics, use once
